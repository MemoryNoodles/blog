IT学习方法集成

1、现在的社会什么都讲求一个快字，那当公司要求用某种语言做某个功能或效果时，应该怎么办？
这个时候我们要做的首先是hold,不要慌不要急。首先想的应该是定方向，比如这门语言那个框架或库比较好用，这样就不用自己去慢慢学基础，自己一点点去搭建，很多复杂、繁琐的工作，库和框架都替你做好了，这样你就可以快速上手一门语言。

2、遇到过的问题，经常会再次遇到，这时你是不是又要花时间去查、搜答案啦？你的积累何在，效率何在
遇到的问题最好记录下来，比如用笔记本、在线云文档、博客等等的形式都是可以的，下次再遇到直接翻自己的博客或文档就可以了（最好记住，这样翻阅博客的时间都省去了），省去网上查找刷选答案的过程。效率是不是又高了啦

3、学习一个新东西的时候，遇到问题怎么办？
遇到的问题会有很多种、比如API的用法、文件的配置、工具的集成、效果的实现、功能的实现…，这个时候一般人都是百度，追求效率的会谷歌。但是其实最快的是在这门语言的官网去search文档这样才是最快的，网上很多人写的东西其实也是去官网转载过来的，而且有时候还很多赞，当然也有人经过自己的理解把一些东西转化成自己的语言表达出来。这时候我们就获得答案的顺序给理一下。首先是官网文档，其次是谷歌。每个人都会搜索，但不是每一个人都能把搜索用好。

mac系统安装指定版本node

发表于 2019-05-07 |

前言

init一个简单demo,运行react-native run-ios的的时候报错，以为是node版本装的太高(因为brew install node自动安装最高版本)，于是想要安装指定版本node,网上找了很多答案都不是很完整，有些甚至很烂。现在写下我的答案。

步骤

1、如果之前使用brew install node安装过node,需要先执行brew unlink node来’解绑’node
2、查找可用的node版本 brew search node
3、安装你需要的版本, 比如 brew reinstall node@8
4、node -v 不出意外, 就安装好了你想要的node版本
5、如果出错请在控制台输入export PATH=”/usr/local/opt/node@8/bin:$PATH”，
6、如果还不行 export LDFLAGS=”-L/usr/local/opt/node@8/lib”
export CPPFLAGS=”-I/usr/local/opt/node@8/include”

Win10和MAC系统通过VMware共享文件夹

发表于 2019-05-06 |

前言

为了react-native能在windows系统上也可以打ios包，于是就在windows系统上创建了虚拟机安装了mac。Win10和MAC系统通过VMware共享文件夹的方法，搞了好久，网上搜的方法都不对，快要放弃之际，突然想到一个办法，希望对你有帮助。

步骤

1、在VMware的Options菜单中选择Shared Folders选项
2、选择Always enabled选项
3、然后选择要在Win10系统上共享的文件夹路径名
4、在Win10系统下将改文件夹设为共享
5、在主机里选中要共享的文件夹，点击上面的共享。
6、进入MAC系统桌面右键点击[前往]菜单中的[连接服务器]选项
7、在连接服务器对话框中输入[smb://Windows主机的IP地址],其中smb是访问Windows共享文件夹所使用的协议名称
8、连接到Windows主机之后会显示该主机所共享的文件夹，选择想要访问的一个即可
9、这时桌面就会有一个共享的文件夹了

结论

高阶组件(HOC)

装饰者(decorator)模式能够在不改变对象自身的基础上，在程序运行期间给对像动态的添加职责。与继承相比，装饰者是一种更轻便灵活的做法。

高阶组件可以看作 React 对装饰模式的一种实现，高阶组件就是一个函数，且该函数接受一个组件作为参数，并返回一个新的组件。

高阶组件（HOC）是 React 中的高级技术，用来重用组件逻辑。但高阶组件本身并不是 React API。它只是一种模式，这种模式是由 React 自身的组合性质必然产生的。

实现方式:

属性代理(Props Proxy)

函数返回一个我们自己定义的组件，然后在 render 中返回要包裹的组件，这样我们就可以代理所有传入的 props，并且决定如何渲染，实际上 ，这种方式生成的高阶组件就是原组件的父组件，上面的函数 visible 就是一个 HOC 属性代理的实现方式。

对比原生组件增强的项：

  • 可操作所有传入的 props
  • 可操作组件的生命周期
  • 可操作组件的 static 方法
  • 获取 refs

反向继承(Inheritance Inversion)

返回一个组件，继承原组件，在 render 中调用原组件的 render。由于继承了原组件，能通过 this 访问到原组件的生命周期、props、state、render 等，相比属性代理它能操作更多的属性。

对比原生组件增强的项：

  • 可操作所有传入的 props
  • 可操作组件的生命周期
  • 可操作组件的 static 方法
  • 获取 refs
  • 可操作 state
  • 可以渲染劫持
